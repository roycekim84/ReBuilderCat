# 냥냥 유적 복원단 상세 기획서

## 1. 문서 목적

이 문서는 **냥냥 유적 복원단(ReBuilderCat)**의 실제 구현을 위한 상세 기획서다.

기존 문서가 큰 방향과 시스템 구조를 잡는 문서라면, 이 문서는 Codex 또는 개발자가 구현 중 세부 판단을 덜 하도록 다음 내용을 구체화한다.

- 게임 시작 흐름
- MVP 플레이 플로우
- 복원 세션의 세부 UX
- 튜토리얼 흐름
- 화면 간 연결
- 팝업/문구 톤
- 우선 구현 기능과 보류 기능

---

## 2. 게임 정체성

### 2.1 게임명

**냥냥 유적 복원단**

### 2.2 프로젝트명

**ReBuilderCat**

### 2.3 장르

- 캐주얼 확률 복원 게임
- 수집형 여행 게임
- 가벼운 역사 탐험 게임

### 2.4 핵심 문장

> 고양이 복원가가 세계 곳곳의 고대 유적을 여행하며, 확률 복원으로 유적을 하나씩 되살리는 게임.

---

## 3. 게임의 핵심 재미

냥냥 유적 복원단의 핵심 재미는 **짧은 판단과 즉각적인 결과**다.

유저는 매번 다음 선택을 한다.

```text
지금 완료해서 안전하게 보상을 받을까?
아니면 한 번 더 복원해서 더 큰 보상을 노릴까?
```

이 구조는 옛날 강화 플래시 게임의 중독성을 가져오지만, 실패를 과하게 벌주는 대신 “유적에 균열이 생겼다” 정도의 귀여운 손상으로 표현한다.

## 4. MVP 전체 플레이 흐름

### 4.1 첫 실행 흐름

1. 앱 실행
2. 타이틀 또는 간단한 로딩
3. 저장 데이터 확인
4. 저장 데이터가 없으면 신규 데이터 생성
5. 튜토리얼 유적 “작은 항구 사당”으로 이동
6. 튜토리얼 안내 시작

### 4.2 일반 실행 흐름

1. 앱 실행
2. 저장 데이터 로드
3. 진행 중인 복원 세션이 있으면 복원 화면 또는 유적 상세 화면 표시
4. 진행 중 세션이 없으면 월드맵 표시

### 4.3 기본 플레이 루프

1. 월드맵에서 유적 선택
2. 유적 상세 화면에서 복원 부위 선택
3. 복원 화면에서 복원하기 시도
4. 성공/실패 결과 확인
5. 완료하기 또는 계속 복원하기 선택
6. 완료 시 보상 획득
7. 유적 복원도 증가
8. 일정 복원도 달성 시 역사 카드 해금
9. 다음 유적 해금

## 5. 튜토리얼 상세

튜토리얼은 1분 안에 끝나야 한다.

### 5.1 튜토리얼 유적

- 유적명: 작은 항구 사당
- 역할: 복원 시스템 설명
- 난이도: 매우 쉬움
- 실패 여부: 첫 복원 시도는 무조건 성공

### 5.2 튜토리얼 단계

#### Step 1. 유적 도착

문구:

```text
냥냥 복원단의 첫 여행지예요!
작은 항구 사당이 오래되어 조금 낡았어요.
```

유저 행동:

- [복원 시작] 버튼 누르기

#### Step 2. 복원하기 안내

문구:

```text
복원하기를 누르면 재료를 사용해 유적을 고칠 수 있어요.
성공하면 복원 단계와 보상이 올라가요!
```

유저 행동:

- [복원하기] 버튼 누르기

결과:

- 무조건 성공

#### Step 3. 완료하기 안내

문구:

```text
지금 완료하면 안전하게 보상을 받을 수 있어요.
하지만 한 번 더 복원하면 보상이 더 커질 수도 있죠!
```

유저 행동:

- 한 번 더 복원하거나 완료하기 선택 가능

#### Step 4. 역사 카드 안내

문구:

```text
유적을 복원하면 역사 카드가 열려요.
여행하며 세계의 이야기를 모아봐요!
```

결과:

- 작은 항구 사당 카드 해금
- 붉은 벽돌 왕국 해금

## 6. 복원 세션 상세 UX

복원 세션은 하나의 복원 부위를 대상으로 한다.

### 6.1 세션 시작 조건

- 해당 유적이 해금되어 있어야 함
- 해당 부위가 해금되어 있어야 함
- 복원 재료가 최소 1회 비용 이상 있어야 함

### 6.2 세션 시작 시 초기값

```text
currentSessionLevel = bestCompletedLevel 또는 0
currentCrackLevel = 0
currentReward = baseReward * 현재 단계 배율
hasActiveSession = true
```

MVP에서는 완료된 부위를 다시 복원할 수 없게 해도 된다.

### 6.3 복원하기 버튼 누를 때

표시 변화:

- 버튼 잠시 비활성화
- 고양이 작업 애니메이션
- 성공/실패 판정
- 결과 문구 표시
- 숫자 업데이트
- 버튼 다시 활성화

### 6.4 성공 시 UX

문구 후보:

```text
복원 성공!
문양이 조금 더 선명해졌어요.
```

```text
좋아요!
고양이가 조심조심 균열을 메웠어요.
```

```text
반짝!
유적이 조금 더 빛나기 시작했어요.
```

효과:

- 복원 단계 +1
- 보상 증가
- 유적 부위 이미지 개선
- 고양이 기쁨 반응

### 6.5 실패 시 UX

문구 후보:

```text
균열 발생!
조금 흔들렸지만 아직 괜찮아요.
```

```text
앗!
돌조각이 살짝 떨어졌어요.
```

```text
고양이가 깜짝 놀랐어요.
다시 조심히 복원해봐요.
```

효과:

- 균열 +1
- 보상 감소
- 단계 하락 가능성은 후반 단계에서만
- 고양이 놀람 반응

### 6.6 완료하기 UX

완료하기를 누르면 현재 상태로 세션을 종료한다.

문구:

```text
복원 완료!
이번 복원으로 유적이 더 안정되었어요.
```

표시:

- 획득 골드
- 획득 재료
- 유적 복원도 증가량
- 별 등급 변화
- 역사 카드 해금 여부

## 7. 실패 복구 광고 UX

### 7.1 표시 조건

- 실패가 발생했을 때
- 유저가 광고 복구 가능 횟수를 초과하지 않았을 때
- 광고 서비스가 사용 가능할 때

### 7.2 팝업 문구

```text
고양이 구조대가 도착했어요!
광고를 보고 이번 균열을 복구할까요?

[그냥 계속하기] [구조대 도움 받기]
```

### 7.3 광고 성공 시

효과 후보:

- 직전 균열 1단계 복구
- 직전 보상 감소 일부 복구
- 단계 하락이 있었다면 단계 복구

MVP에서는 단순하게 다음 중 하나로 처리한다.

```text
crackLevel = max(0, crackLevel - 1)
```

## 8. 유적 완료 UX

유적은 60% 이상 복원하면 클리어된다.

### 8.1 1성 클리어

조건:

- 복원도 60% 이상

문구:

```text
유적 복원 완료!
다음 여행지로 떠날 수 있어요.
```

### 8.2 2성 우수 복원

조건:

- 복원도 80% 이상

문구:

```text
멋진 복원이에요!
유적이 예전 모습을 많이 되찾았어요.
```

### 8.3 3성 완전 복원

조건:

- 복원도 100%

문구:

```text
완전 복원 성공!
냥냥 복원단의 이름이 지도에 기록되었어요.
```

## 9. 역사 카드 UX

역사 카드는 복원 보상처럼 제공한다.

### 9.1 해금 타이밍

- 유적 60% 이상 복원 시 기본 카드 해금
- 100% 복원 시 완전 복원 표시 추가

### 9.2 카드 표시 방식

유적 완료 팝업 이후:

```text
새 역사 카드 획득!
[카드 보기] [나중에 보기]
```

### 9.3 카드 문체

- 3~5문장
- 너무 딱딱하지 않게
- 실제 역사 기반
- 마지막 문장은 게임 세계관과 연결 가능

예시:

```text
붉은 벽돌 왕국
영감: 아유타야 유적

강가에 세워진 오래된 왕국의 흔적이에요.
붉은 벽돌탑과 사원은 옛 도시의 번영을 보여줘요.
냥냥 복원단은 무너진 탑의 길을 다시 정리했어요.
```

## 10. 월드맵 상세 흐름

### 10.1 월드맵 기본 구조

- 파트 단위로 유적 노드 표시
- 선형 진행 중심
- 일부 유적은 나중에 분기 가능

MVP에서는 선형 구조로 시작한다.

```text
작은 항구 사당 → 붉은 벽돌 왕국 → 안개의 계단 사원 → 숲속 고대 사원 → 정글의 붉은 탑 → 바위 왕국 유적
```

### 10.2 해금 조건

다음 유적은 이전 유적 1성 이상 클리어 시 해금한다.

```text
previousRelic.starCount >= 1
```

### 10.3 잠김 문구

```text
이 유적은 아직 찾지 못했어요.
이전 유적을 복원하면 지도가 이어져요.
```

## 12. 고양이 캐릭터 사용 방식

고양이는 단순 장식이 아니라 피드백 담당이다.

### 12.1 기본 반응

| 상황 | 고양이 반응 |
|---|---|
| 대기 | 붓을 들고 바라봄 |
| 복원 중 | 집중해서 톡톡 두드림 |
| 성공 | 만세 또는 반짝 눈 |
| 실패 | 깜짝 놀람 |
| 재료 부족 | 빈 가방을 봄 |
| 유적 완료 | 지도에 체크 |
| 역사 카드 | 책을 읽음 |

### 12.2 구현 초기 방식

초기에는 실제 애니메이션 대신 상태별 이미지 또는 이모지/아이콘 대체 가능.

단, 코드 구조는 나중에 Sprite/Image asset으로 교체하기 쉽게 만든다.

## 15. MVP 완료 기준

MVP는 다음 흐름이 처음부터 끝까지 가능하면 완료로 본다.

```text
앱 실행
→ 작은 항구 사당 선택
→ 복원 부위 선택
→ 복원하기 성공/실패
→ 완료하기
→ 보상 획득
→ 유적 복원도 증가
→ 역사 카드 해금
→ 다음 유적 해금
→ 도감에서 카드 확인
→ 상점에서 보급품 받기
```

## 16. 상세 기획 최종 정리

냥냥 유적 복원단은 기능이 많은 게임이 아니라, 같은 행동이 기분 좋게 반복되는 게임이다.

따라서 구현 중 항상 다음 기준을 우선한다.

1. 복원하기 버튼이 명확한가?
2. 성공/실패 결과가 즉시 느껴지는가?
3. 실패해도 너무 짜증나지 않는가?
4. 유적이 점점 좋아지고 있다는 느낌이 있는가?
5. 고양이가 귀엽게 반응하는가?
6. 다음 여행지로 가고 싶어지는가?

이 기준을 만족하면 MVP는 작아도 충분히 게임답다.
