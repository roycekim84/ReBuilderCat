# 냥냥 유적 복원단 이미지/에셋 기획서

## 1. 문서 목적

이 문서는 **냥냥 유적 복원단(ReBuilderCat)**의 이미지, UI 에셋, 캐릭터, 유적, 배경, 아이콘 제작 방향을 정의한다.

MVP 구현 초반에는 placeholder로 시작하되, 최종적으로 어떤 에셋이 필요한지 흔들리지 않도록 기준을 정리한다.

---

## 2. 전체 아트 방향

### 2.1 핵심 분위기

- 귀여운 고양이 탐험대
- 고대 유적 복원
- 세계여행
- 따뜻한 그림책 감성
- 캐주얼 모바일 게임

### 2.2 권장 스타일

아트 스타일은 다음 둘 중 하나를 선택할 수 있다.

#### A안. 귀여운 2D 일러스트

- 구현/가독성 좋음
- 모바일 UI와 잘 어울림
- 에셋 재사용 쉬움
- 추천도 높음

#### B안. 픽셀아트 감성

- 플래시 게임/레트로 감성 강함
- 강화 게임 느낌과 잘 맞음
- 다만 유적 단계별 에셋 제작량이 늘 수 있음

MVP 권장:

> 귀여운 2D 일러스트 기반으로 가되, UI와 아이콘에 픽셀/도트 감성을 살짝 섞는다.

## 3. 화면 비율과 해상도 기준

### 3.1 기본 기준

- 모바일 세로 화면 기준
- 기준 해상도: 1080 x 1920
- 실제 에셋은 다양한 해상도 대응을 고려해 여유 있게 제작

### 3.2 주요 에셋 권장 크기

| 에셋 | 권장 크기 | 비고 |
|---|---:|---|
| 캐릭터 기본 스프라이트 | 512 x 512 | 상태별 단일 이미지 가능 |
| 유적 대표 이미지 | 1024 x 768 | 유적 상세 화면 |
| 복원 부위 이미지 | 768 x 768 | 복원 화면 중앙 |
| 배경 이미지 | 1080 x 1200 | 상단~중앙 배경 |
| 월드맵 이미지 | 1080 x 1400 | 세로 지도 |
| 역사 카드 이미지 | 800 x 600 | 카드 썸네일 |
| 아이콘 | 256 x 256 | 재료/골드/아이템 |
| 앱 아이콘 | 1024 x 1024 | 스토어용 |

## 4. 고양이 캐릭터 기획

### 4.1 캐릭터 정체성

고양이는 유저의 분신이자 게임의 감정 전달자다.

컨셉:

- 작은 고양이 복원가
- 탐험 모자 또는 작은 스카프 착용
- 붓, 작은 망치, 돋보기, 지도 가방 사용
- 너무 전문적인 고고학자보다 귀여운 초보 복원가 느낌

### 4.2 기본 외형

- 둥근 얼굴
- 작은 몸
- 짧은 다리
- 큰 눈
- 귀여운 꼬리
- 따뜻한 크림색/흰색/연갈색 계열 권장
- 복잡한 무늬는 최소화

### 4.3 필요한 상태 이미지

MVP 필수:

| 상태 | 용도 | 우선순위 |
|---|---|---:|
| 기본 대기 | 대부분 화면 | 필수 |
| 복원 중 | 복원하기 버튼 후 | 필수 |
| 성공 | 복원 성공 | 필수 |
| 실패/놀람 | 균열 발생 | 필수 |
| 완료 기쁨 | 부위/유적 완료 | 필수 |
| 지도 보기 | 월드맵 | 추천 |
| 카드 읽기 | 역사 카드 | 추천 |
| 재료 부족 | 상점 유도 | 추천 |

MVP 최소로는 5장만 있어도 된다.

### 4.4 추후 의상 확장

꾸미기 업데이트용 후보:

- 기본 탐험가 모자
- 사막 여행자 망토
- 고대 학자 로브
- 정글 탐험 조끼
- 눈 덮인 유적 털모자
- 별빛 복원가 의상

## 5. 유적 에셋 기획

### 5.1 유적 표현 원칙

- 실제 유적을 정밀 복제하지 않는다.
- 특징은 살리되 게임 내 이름과 디자인을 변형한다.
- 공식 사진, 로고, 유네스코 마크 등은 사용하지 않는다.
- 직접 제작한 오리지널 스타일 이미지로 표현한다.

### 5.2 유적 단계별 변화

유적은 복원도에 따라 변화해야 한다.

공통 단계:

| 단계 | 상태 | 이미지 특징 |
|---|---|---|
| 0 | 낡음 | 먼지, 이끼, 균열, 흐린 색감 |
| 1 | 정리됨 | 먼지 일부 제거, 윤곽 확인 |
| 2 | 수리됨 | 큰 균열 감소, 구조 안정 |
| 3 | 복원 중 | 문양과 장식 일부 복구 |
| 4 | 완성 | 밝은 색감, 반짝임, 고양이 깃발 |
| 5 | 완전 복원 | 추가 빛 효과, 별 장식, 완성 마크 |

MVP에서는 모든 유적에 6단계 이미지를 만들 필요는 없다.

초기 구현 방식:

- 유적 대표 이미지는 3단계만 준비 가능
  - 낡음
  - 중간 복원
  - 완성
- 복원 부위는 색상/오버레이/아이콘으로 단계 표현 가능

## 7. 월드맵 이미지 기획

### 7.1 MVP 월드맵

MVP는 아시아~남아시아 초입까지만 표현한다.

구성:

```text
작은 시작섬
  ↓
붉은 벽돌 왕국
  ↓
안개의 계단 사원
  ↓
숲속 고대 사원
  ↓
정글의 붉은 탑
  ↓
바위 왕국 유적
```

### 7.2 지도 스타일

- 낡은 양피지 지도 느낌
- 귀여운 고양이 발자국 경로
- 각 유적은 둥근 노드 또는 작은 미니 일러스트
- 잠긴 유적은 구름/안개로 가림

### 7.3 지도 아이콘

필요 아이콘:

- 현재 위치 고양이 머리 아이콘
- 완료 체크
- 별 1~3개
- 잠금 아이콘
- 다음 목적지 반짝임

## 12. 앱 아이콘 기획

### 12.1 핵심 요소

- 고양이 얼굴
- 작은 붓 또는 망치
- 유적 조각
- 반짝임

### 12.2 방향

- 1024x1024
- 텍스트 없음
- 작은 크기에서도 고양이와 복원 느낌이 보여야 함
- 배경은 따뜻한 모래색 또는 하늘색

아이콘 컨셉 예시:

> 탐험 모자를 쓴 고양이가 작은 붓으로 깨진 유적 조각을 반짝이게 복원하는 모습.

## 16. 금지사항

- 실제 유적 사진을 그대로 사용하지 않는다.
- 공식 관광청/박물관 이미지 무단 사용 금지
- 유네스코 로고 사용 금지
- 특정 현대 건축물/브랜드 건물을 정밀 묘사하지 않는다.
- UI에 너무 많은 장식을 넣어 가독성을 해치지 않는다.
- 고양이 캐릭터를 지나치게 복잡하게 만들지 않는다.

---

## 17. 최종 방향

이미지 기획의 핵심은 “귀엽고 읽기 쉬운 복원 감성”이다.

유적은 웅장하지만 어렵지 않아야 하고, 고양이는 작지만 화면의 감정을 이끌어야 한다. 에셋이 완벽하지 않아도 복원 단계의 변화와 성공/실패 피드백이 느껴지면 게임은 충분히 살아난다.
