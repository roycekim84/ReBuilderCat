# ReBuilderCat 컨셉 기획서

## 1. 프로젝트 개요

**프로젝트명:** ReBuilderCat  
**가칭 한국어명:** 냥냥 유적 복원단 / 고양이 유적 복원가 / 리빌더캣  
**장르:** 캐주얼 확률 복원 게임, 수집형 여행 게임, 가벼운 역사 탐험 게임  
**플랫폼:** 모바일 우선, Android 우선 출시, 이후 iOS 확장 가능  
**개발 방향:** Flutter 기반의 단순한 버튼 중심 모바일 게임  

ReBuilderCat은 귀여운 고양이 복원가가 세계 곳곳의 고대 유적을 여행하며, 확률 기반 복원 시스템으로 유적의 부위를 하나씩 되살리는 캐주얼 게임이다. 옛날 플래시 게임의 “강화 버튼을 누르고, 성공 확률을 보며, 적당한 타이밍에 멈추는 재미”를 기반으로 하되, 검이나 무기 강화 대신 “유적 복원”이라는 따뜻한 목표를 부여한다.

핵심은 매우 단순하다.

> 복원할까, 지금 완료할까?

유저는 복원 성공률, 현재 보상, 다음 성공 시 보상, 균열 위험을 보며 한 번 더 도전할지 고민한다. 성공하면 유적이 조금 더 아름답게 복원되고 보상이 증가한다. 실패하면 균열이 생기거나 복원도가 일부 감소하지만, 게임 전체가 너무 고단하지 않도록 완전 파괴나 극단적인 초기화는 최소화한다.

---

## 2. 핵심 한 줄 컨셉

> 고양이 복원가가 세계의 고대 유적을 여행하며, 확률 복원으로 유적을 하나씩 되살리는 캐주얼 탐험 게임.

---

## 3. 기획 의도

ReBuilderCat은 복잡한 조작 없이도 짧게 반복 플레이할 수 있는 게임을 목표로 한다. 버튼을 누르는 즉각적인 재미, 성공/실패의 긴장감, 여행지를 하나씩 넓혀가는 성장감, 복원된 유적을 바라보는 만족감이 핵심이다.

기존 강화 게임의 중독성은 유지하되, 분위기는 더 귀엽고 따뜻하게 만든다. 유저가 실패했을 때도 “터졌다”는 느낌보다는 “조금 손상됐지만 다시 고쳐보자”는 느낌이 들게 한다.

또한 실제 고대 유적과 문명에서 영감을 얻은 지역을 순서대로 여행하면서, 짧은 역사 카드와 도감 요소를 통해 부담 없는 역사 공부도 제공한다. 공부를 강제하지 않고, 복원 완료 보상처럼 자연스럽게 열리는 방식으로 접근한다.

## 4. 목표 유저

### 4.1 주 타깃

- 귀여운 고양이 캐릭터를 좋아하는 유저
- 짧은 시간에 반복 플레이하는 캐주얼 게임 유저
- 확률 강화, 수집, 도감 채우기 요소를 좋아하는 유저
- 세계여행, 유적, 역사 소재에 가볍게 흥미가 있는 유저

### 4.2 플레이 상황

- 출퇴근길에 1~3분 플레이
- 쉬는 시간에 복원 몇 번 시도
- 자기 전 월드맵 진행도 확인
- 광고 보상으로 실패 복구 또는 재료 보급 받기

---

## 5. 게임의 핵심 감정

ReBuilderCat이 유저에게 주어야 하는 감정은 다음과 같다.

1. **한 번만 더 누르고 싶은 긴장감**  
   성공 확률이 조금 낮아졌지만, 다음 성공 보상이 매력적이어야 한다.

2. **유적이 살아나는 만족감**  
   단순히 숫자만 오르는 것이 아니라, 복원 단계에 따라 유적 이미지가 조금씩 깨끗해지고 아름다워져야 한다.

3. **고양이와 함께 여행하는 감성**  
   월드맵에서 새로운 지역이 열리고, 고양이가 작은 배나 지도 위를 이동하는 느낌을 준다.

4. **짧게 배우는 역사 발견감**  
   유적을 완료하면 역사 카드가 열리고, 유저가 부담 없이 읽을 수 있는 짧은 지식이 제공된다.

5. **실패해도 다시 할 수 있는 부드러운 스트레스**  
   실패는 있어야 하지만 유저가 앱을 지우고 싶을 정도로 강하면 안 된다.

## 6. 게임 루프

### 6.1 기본 루프

1. 월드맵에서 유적 선택
2. 유적 상세 화면에서 복원할 부위 선택
3. 메인 복원 화면에서 복원 시도
4. 성공 시 복원 단계 상승, 보상 증가, 유적 비주얼 개선
5. 실패 시 균열 발생, 일부 보상 감소 또는 복원도 감소
6. 유저가 적당한 타이밍에 완료하기 선택
7. 보상 획득, 유적 전체 복원도 증가
8. 일정 복원도 달성 시 다음 부위 또는 다음 유적 해금
9. 유적 완료 시 역사 카드 획득
10. 다음 여행지로 이동

### 6.2 핵심 선택

유저는 매 시도마다 다음 둘 중 하나를 선택한다.

- **복원하기:** 위험을 감수하고 더 높은 보상과 완성도를 노린다.
- **완료하기:** 현재 상태로 안전하게 보상을 받고 복원을 마무리한다.

이 선택이 게임의 핵심 재미다.

## 7. 복원 시스템 개요

### 7.1 복원 단계

각 복원 부위는 여러 단계로 구성된다. 초반 기준으로 부위당 4~5단계가 적당하다.

예시:

| 단계 | 상태 | 설명 |
|---|---|---|
| 0단계 | 낡음 | 먼지와 균열이 많은 상태 |
| 1단계 | 먼지 제거 | 기본 정리 완료 |
| 2단계 | 균열 수리 | 형태가 안정됨 |
| 3단계 | 문양 복원 | 유적의 특징이 살아남 |
| 4단계 | 복원 완료 | 반짝이는 완성 상태 |

### 7.2 성공 확률

초반에는 성공 확률을 높게 시작해 접근성을 확보한다.

예시:

| 복원 단계 | 성공률 예시 |
|---|---:|
| +1 | 95% |
| +2 | 85% |
| +3 | 72% |
| +4 | 58% |
| +5 | 45% |

후반 유적은 성공률이 조금 낮아질 수 있지만, 너무 극단적인 실패율은 지양한다.

### 7.3 실패 처리

실패는 “파괴”가 아니라 “손상” 또는 “균열”로 표현한다.

가능한 실패 효과:

- 현재 보상 일부 감소
- 균열 게이지 증가
- 복원 단계 1 하락
- 재료 일부 소모
- 고양이 반응 애니메이션 출력

지양할 실패 효과:

- 잦은 완전 초기화
- 잦은 완전 파괴
- 유저가 긴 플레이 시간을 한 번에 모두 잃는 구조

---

## 8. 복원 완료와 보상

유저는 복원 중 언제든 “완료하기”를 눌러 현재 상태로 보상을 받을 수 있다.

완료 보상은 다음 요소에 따라 결정된다.

- 현재 복원 단계
- 연속 성공 횟수
- 유적 난이도
- 사용한 보조 아이템
- 이벤트 보너스 여부

유적 전체 진행도는 부위별 완료 상태를 기반으로 오른다.

권장 구조:

- 60% 복원: 유적 클리어 가능
- 80% 복원: 추가 보상
- 100% 복원: 별 3개, 역사 카드 완전 해금, 도감 완성

이렇게 하면 캐주얼 유저는 지나갈 수 있고, 수집형 유저는 100% 완성을 노릴 수 있다.

## 9. 여행/파트 구조

게임은 세계 곳곳을 파트 단위로 확장한다. 시작은 아시아에서 하고, 이후 점점 멀리 여행하는 구조가 좋다.

### 9.1 전체 파트 흐름

1. **Part 1. 아시아의 첫 유적**
2. **Part 2. 인도와 남아시아**
3. **Part 3. 사막과 중동의 고대 도시**
4. **Part 4. 지중해와 유럽 고대 문명**
5. **Part 5. 아프리카의 왕국과 사원**
6. **Part 6. 아메리카 고대 문명**
7. **Part 7. 환상의 고대 유적 / 엔드 콘텐츠**

### 9.2 출시 초기 추천 범위

출시 전 완성 목표는 너무 크게 잡지 않는다.

추천 초기 구성:

- 튜토리얼 유적 1개
- 아시아 파트 유적 4개
- 인도/남아시아 맛보기 유적 1~2개

총 5~7개 유적이면 첫 출시 버전으로 충분하다.

## 10. 유적 후보 방향

실제 고대 유적에서 영감을 얻되, 게임 내 명칭은 살짝 변형해도 된다. 법적/문화적 리스크를 줄이고, 게임 고유의 세계관을 만들기 위해 실제 명칭과 판타지 명칭을 혼합한다.

### 10.1 Part 1. 아시아의 첫 유적

| 실제 영감 | 게임 내 이름 예시 | 특징 |
|---|---|---|
| 작은 항구 사당 | 작은 항구 사당 | 튜토리얼 유적 |
| 아유타야 | 붉은 벽돌 왕국 | 벽돌탑과 강가 도시 느낌 |
| 보로부두르 | 안개의 계단 사원 | 단계형 복원 연출에 적합 |
| 앙코르 | 숲속 고대 사원 | 정글, 이끼, 거대 석조 사원 |
| 미선 유적 | 정글의 붉은 탑 | 붉은 벽돌과 숲 배경 |

### 10.2 Part 2. 인도와 남아시아

| 실제 영감 | 게임 내 이름 예시 | 특징 |
|---|---|---|
| 함피 | 바위 왕국 유적 | 거대한 돌기둥, 왕국 폐허 |
| 산치 스투파 | 둥근 성스러운 탑 | 원형 구조, 평온한 분위기 |
| 시기리야 | 하늘 바위 궁전 | 높은 바위와 궁전 느낌 |

### 10.3 Part 3 이후 확장 후보

- 장밋빛 바위 도시
- 태양의 삼각 무덤
- 흰 기둥의 언덕 신전
- 고대 원형 경기장
- 구름 위 돌도시
- 깃털뱀의 계단 신전
- 정글의 높은 신전

## 11. 역사 카드 시스템

유적을 복원하면 짧은 역사 카드가 열린다. 역사 카드는 게임의 교육 요소이자 수집 요소다.

### 11.1 원칙

- 강제로 읽히지 않는다.
- 3줄 내외로 짧게 제공한다.
- 어린 유저도 이해할 수 있게 쉽게 쓴다.
- 실제 사실 기반으로 작성하되, 게임 내 이름이 변형된 경우 “영감을 받은 실제 유적” 정보를 별도로 제공할 수 있다.

### 11.2 카드 구성

- 유적 이름
- 지역/나라
- 시대 또는 문명
- 짧은 설명
- 재미있는 포인트 1개
- 복원 달성도

### 11.3 예시

```text
숲속 고대 사원
영감: 앙코르 유적

오래전 거대한 왕국 사람들이 만든 숲속의 사원이에요.
돌기둥과 벽에는 당시 사람들의 믿음과 생활이 새겨져 있어요.
시간이 지나 숲에 묻혔지만, 지금도 웅장한 모습을 간직하고 있어요.
```

## 12. 화면 구성 MVP

출시 전 기준으로 5개 화면을 우선 완성한다.

### 12.1 메인 복원 화면

게임의 핵심 플레이 화면이다.

표시 요소:

- 현재 유적 배경
- 고양이 복원가 캐릭터
- 복원 중인 유적 부위 이미지
- 현재 복원 단계
- 성공 확률
- 균열 위험
- 현재 보상
- 다음 성공 시 예상 보상
- 복원하기 버튼
- 완료하기 버튼
- 보유 재료
- 보유 골드

### 12.2 월드맵 / 여행 화면

파트와 유적을 선택하는 화면이다.

표시 요소:

- 현재 파트 이름
- 지도 경로
- 유적 노드
- 별 달성도
- 다음 유적 잠금 상태
- 고양이 이동 연출
- 보급품 받기 버튼

### 12.3 유적 상세 화면

선택한 유적의 복원 진행도를 보여주는 화면이다.

표시 요소:

- 유적명
- 전체 복원도
- 복원 부위 목록
- 각 부위 상태: 잠김, 진행 가능, 완료
- 대표 유적 이미지
- 복원 시작 버튼

### 12.4 도감 / 역사 카드 화면

수집한 유적 정보와 역사 카드를 확인하는 화면이다.

표시 요소:

- 유적 목록
- 역사 카드 상세
- 짧은 설명
- 수집률
- 별 달성도

### 12.5 상점 / 보급품 화면

재료, 광고 보상, 광고 제거 상품을 제공하는 화면이다.

표시 요소:

- 매일 무료 보급품
- 광고 보고 보급품 받기
- 보조 아이템 구매
- 광고 제거 상품
- 재화 잔액

## 14. BM 방향

ReBuilderCat은 강한 과금 게임이 아니라, 귀엽고 가벼운 광고 기반 캐주얼 게임을 지향한다.

### 14.1 출시 초기 BM

필수 BM:

1. 보상형 광고 - 실패 후 균열 복구
2. 보상형 광고 - 재료 보급품 받기
3. 인앱결제 - 광고 제거

선택 BM:

4. 유적 완료 후 낮은 빈도의 전면 광고

### 14.2 권장 표현

상업적인 표현보다 게임 세계관에 맞는 표현을 사용한다.

| 일반 표현 | 게임 내 표현 |
|---|---|
| 광고 보고 보상 | 탐험대 지원 받기 |
| 광고 보고 복구 | 구조대 도움 받기 |
| 광고 보고 재료 | 보급품 받기 |
| 광고 제거 | 후원자 패스 |
| 상점 | 탐험 지원소 |
| 패키지 | 고양이 탐험 후원팩 |

### 14.3 주의점

- 복원 버튼 몇 번마다 강제 광고를 넣지 않는다.
- 실패할 때마다 강제 광고를 띄우지 않는다.
- 도감/역사 카드 화면에는 광고 요소를 최소화한다.
- 초반 평점을 위해 전면 광고는 보류하거나 매우 낮은 빈도로 사용한다.

## 15. 아트 방향

### 15.1 스타일

- 귀여운 2D 일러스트 또는 픽셀아트 감성
- 고양이는 작고 둥글고 사랑스러운 비율
- 유적은 실제 유적을 정밀 복제하기보다 특징만 살린 게임풍 표현
- 과도한 사실주의보다 따뜻한 탐험 그림책 느낌

### 15.2 고양이 캐릭터

고양이는 게임의 감정 전달자다.

필요한 기본 상태:

- 기본 대기
- 복원 시도
- 성공 기쁨
- 실패 놀람
- 균열 발생 울상
- 역사 카드 읽기
- 월드맵 이동

### 15.3 유적 비주얼

유적은 단계별로 변화해야 한다.

예시:

- 0단계: 먼지, 이끼, 균열
- 1단계: 먼지 제거
- 2단계: 큰 균열 수리
- 3단계: 문양 복원
- 4단계: 빛과 장식 추가

- Hive 또는 SharedPreferences
- AdMob

- Android 우선 출시

### 17.2 엔진 사용 방향

MVP에서는 순수 Flutter 중심으로 구현한다. Flame은 필수로 도입하지 않는다.

이유:

- 게임 조작이 버튼 중심이다.
- 화면 수가 적다.
- UI와 상태 관리가 중요하다.
- Codex 작업이 단순해진다.
- Flutter 애니메이션만으로도 충분히 손맛을 만들 수 있다.

향후 이펙트, 파티클, 캐릭터 이동 연출이 많아질 경우 Flame 도입을 재검토한다.

## 20. 최종 방향성 정리

ReBuilderCat은 복잡한 경영 게임이나 액션 게임이 아니다. 핵심은 단순한 버튼 플레이와 짧은 확률 판단이다. 하지만 그 단순함 위에 고양이, 유적 복원, 세계여행, 역사 카드라는 감성을 얹어 오래 기억되는 캐주얼 게임으로 만든다.

개발 중 우선순위는 다음과 같다.

1. 복원 버튼의 손맛
2. 성공/실패 피드백
3. 유적이 복원되는 시각적 만족감
4. 월드맵의 여행감
5. 역사 카드의 수집 보상감
6. 광고/BM의 자연스러운 배치

이 문서의 기준은 “작게 만들고 실제로 출시한다”이다. 모든 추가 기능은 이 기준을 해치지 않는 범위에서만 넣는다.
