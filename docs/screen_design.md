# 냥냥 유적 복원단 화면 설계서

## 1. 문서 목적

이 문서는 **냥냥 유적 복원단(ReBuilderCat)**의 출시 전 MVP 기준 5개 핵심 화면을 구현 가능한 수준으로 정리한다.

목표는 화면 수를 최소화하면서도 다음 감정을 확실히 전달하는 것이다.

- 고양이와 함께 세계를 여행하는 느낌
- 유적을 단계별로 복원하는 만족감
- 복원 버튼을 누르는 짧고 강한 손맛
- 역사 카드와 도감 수집의 보상감
- 광고/BM이 자연스럽게 들어갈 수 있는 구조

MVP 기준 화면은 다음 5개다.

1. 메인 복원 화면
2. 월드맵 / 여행 화면
3. 유적 상세 화면
4. 도감 / 역사 카드 화면
5. 상점 / 보급품 화면

추후 추가 화면은 다음 2개다.

1. 고양이 꾸미기 화면
2. 미션 / 업적 화면

---

## 2. 공통 UI 원칙

### 2.1 화면 방향

- 모바일 세로 화면 기준
- 한 손 조작 가능해야 함
- 핵심 버튼은 하단 영역에 크게 배치
- 정보는 많아 보여도 실제 선택지는 적게 유지

### 2.2 분위기

- 귀여운 고양이 탐험대 느낌
- 고대 유적 + 그림책 + 캐주얼 게임 분위기
- 너무 어둡거나 무겁지 않게 표현
- 실패해도 좌절보다 귀여운 해프닝처럼 보이게 처리

### 2.3 UI 톤

권장 표현:

- 강화하기 → 복원하기
- 판매하기 → 완료하기
- 실패 → 균열 발생
- 파괴 → 손상
- 광고 보상 → 탐험대 지원
- 상점 → 탐험 지원소
- 아이템 → 복원 도구 / 보급품

---

## 3. 화면 1: 메인 복원 화면

## 3.1 역할

게임의 핵심 플레이 화면이다. 전체 플레이 시간의 대부분이 이 화면에서 발생한다.

유저는 현재 복원 부위의 성공률과 위험도를 보고, **복원하기** 또는 **완료하기**를 선택한다.

핵심 감정:

> 지금 완료하면 안전하지만, 한 번 더 복원하면 더 큰 보상을 받을 수 있다.

---

## 3.2 진입 경로

- 유적 상세 화면에서 복원 부위 선택 후 진입
- 실패 복구 후 다시 진입
- 앱 재실행 시 진행 중인 복원이 있다면 이 화면으로 복귀 가능

---

## 3.3 화면 구성

```text
┌─────────────────────────┐
│ 상단바: 골드 / 재료 / 설정 │
├─────────────────────────┤
│ 현재 유적명                │
│ 예: 숲속 고대 사원          │
├─────────────────────────┤
│ 유적 배경 이미지            │
│                           │
│        복원 부위           │
│       +3 돌기둥            │
│                           │
│   고양이 복원가 애니메이션   │
├─────────────────────────┤
│ 성공률 72%                 │
│ 균열 위험 12%              │
│ 현재 보상 1,200            │
│ 다음 성공 보상 1,850        │
├─────────────────────────┤
│ [복원하기] [완료하기]        │
└─────────────────────────┘
```

---

## 3.4 표시 정보

### 필수 표시

- 현재 유적 이름
- 현재 복원 부위 이름
- 현재 복원 단계
- 성공률
- 균열 위험
- 현재 완료 보상
- 다음 성공 시 예상 보상
- 보유 골드
- 보유 복원 재료
- 복원하기 버튼
- 완료하기 버튼

### 선택 표시

- 연속 성공 횟수
- 사용 중인 보조 아이템
- 오늘 남은 광고 복구 횟수
- 고양이 상태 문구

---

## 3.5 주요 버튼

### 복원하기

- 복원 시도 실행
- 재료 소모
- 성공/실패 판정
- 결과 애니메이션 재생

버튼 문구 예시:

- 복원하기
- 조심조심 복원!
- 한 번 더 복원!

MVP에서는 **복원하기**로 고정한다.

### 완료하기

- 현재 복원 상태로 마무리
- 보상 지급
- 유적 상세 화면으로 이동

버튼 문구 예시:

- 완료하기
- 여기까지 완료
- 안전하게 완료

MVP에서는 **완료하기**로 고정한다.

---

## 3.6 성공 결과 연출

성공 시:

- 복원 부위가 반짝임
- 복원 단계 +1
- 고양이가 기뻐함
- 보상 숫자가 증가하는 애니메이션
- 성공 사운드 재생

문구 예시:

```text
복원 성공!
돌기둥의 문양이 조금 더 선명해졌어요.
```

---

## 3.7 실패 결과 연출

실패 시:

- 유적 부위에 작은 균열 이펙트
- 고양이가 놀람
- 보상 일부 감소 또는 단계 하락
- 실패 사운드 재생
- 필요 시 광고 복구 팝업 표시

문구 예시:

```text
균열 발생!
복원 부위가 조금 흔들렸어요.
```

실패는 절대 과하게 무섭게 표현하지 않는다.

---

## 3.8 광고 복구 팝업

실패 후 일정 조건에서 표시한다.

```text
고양이 구조대가 도와줄 수 있어요!
광고를 보고 이번 균열을 복구할까요?

[그냥 계속하기] [구조대 도움 받기]
```

조건:

- 부위당 1회 또는 하루 제한
- 모든 실패에 강제로 띄우지 않음
- 닫기 쉬워야 함

---

## 3.9 상태별 고양이 반응

| 상태 | 반응 |
|---|---|
| 기본 | 붓을 들고 대기 |
| 복원 중 | 땀 흘리며 집중 |
| 성공 | 만세 / 반짝 눈 |
| 실패 | 깜짝 놀람 |
| 완료 | 지도에 체크 표시 |
| 재료 부족 | 가방을 뒤적임 |

---

## 3.10 빈 상태 / 예외 처리

### 재료 부족

```text
복원 재료가 부족해요.
탐험 지원소에서 보급품을 받을 수 있어요.

[탐험 지원소로] [광고 보고 보급품]
```

### 이미 완료된 부위

- 유적 상세 화면으로 돌려보냄
- “이미 완료한 복원 부위예요.” 표시

---

## 4. 화면 2: 월드맵 / 여행 화면

## 4.1 역할

세계 여행과 진행도를 보여주는 화면이다. 유저가 현재 어느 파트에 있고, 다음 목적지가 어디인지 이해할 수 있어야 한다.

핵심 감정:

> 고양이와 함께 세계 지도를 따라 점점 먼 유적지로 떠난다.

---

## 4.2 진입 경로

- 앱 메인 진입 시 기본 화면 후보
- 유적 완료 후 복귀
- 하단 네비게이션에서 월드맵 선택

---

## 4.3 화면 구성

```text
┌─────────────────────────┐
│ 상단바: 골드 / 재료        │
├─────────────────────────┤
│ Part 1. 아시아의 첫 유적   │
│ 진행도 3 / 5              │
├─────────────────────────┤
│         지도 이미지        │
│   ● 작은 항구 사당          │
│      ↓                    │
│   ● 붉은 벽돌 왕국          │
│      ↓                    │
│   ● 안개의 계단 사원        │
│      ↓                    │
│   ○ 숲속 고대 사원          │
│      ↓                    │
│   🔒 정글의 붉은 탑          │
├─────────────────────────┤
│ [선택한 유적 정보 카드]     │
│ [입장하기]                 │
└─────────────────────────┘
```

---

## 4.4 표시 정보

- 현재 파트명
- 파트 진행도
- 유적 노드 목록
- 유적별 클리어 상태
- 별 달성도
- 잠금 조건
- 선택한 유적 미리보기
- 입장하기 버튼

---

## 4.5 유적 노드 상태

| 상태 | 표시 |
|---|---|
| 잠김 | 자물쇠 |
| 입장 가능 | 밝은 노드 |
| 진행 중 | 고양이 발자국 |
| 완료 | 체크 표시 |
| 100% 완료 | 별 3개 |

---

## 4.6 파트 이동

초기 MVP에서는 Part 1 중심으로 구현한다.

- 좌우 스와이프 또는 파트 탭으로 이동 가능
- 잠긴 파트는 미리보기만 제공
- “다음 업데이트 예정” 표현 가능

MVP에서는 실제 플레이 가능 파트만 열고, 나머지는 비활성 표시해도 된다.

---

## 4.7 월드맵 연출

- 유적 해금 시 고양이가 지도 위를 이동
- 발자국 애니메이션
- 작은 배/기차/비행기 아이콘은 추후 추가 가능
- 파트 완료 시 지도에 반짝 효과

---

## 5. 화면 3: 유적 상세 화면

## 5.1 역할

선택한 유적의 복원 진행도를 보여주고, 복원할 부위를 선택하는 화면이다.

핵심 감정:

> 이 유적이 조금씩 되살아나고 있다.

---

## 5.2 진입 경로

- 월드맵에서 유적 선택
- 복원 완료 후 복귀
- 도감에서 해당 유적 보기 선택 시 읽기 전용으로 진입 가능

---

## 5.3 화면 구성

```text
┌─────────────────────────┐
│ 숲속 고대 사원             │
│ 전체 복원도 45%            │
├─────────────────────────┤
│       대표 유적 이미지      │
│   복원 단계에 따라 변화     │
├─────────────────────────┤
│ 복원 부위                  │
│ ✅ 입구 계단 완료           │
│ ▶ 돌기둥 진행 가능          │
│ 🔒 벽화 잠김                │
│ 🔒 중앙 제단 잠김            │
├─────────────────────────┤
│ [복원 시작]                │
└─────────────────────────┘
```

---

## 5.4 표시 정보

- 유적명
- 지역/파트명
- 전체 복원도
- 대표 이미지
- 부위 목록
- 부위별 상태
- 예상 보상
- 역사 카드 해금 여부
- 복원 시작 버튼

---

## 5.5 복원 부위 상태

| 상태 | 설명 |
|---|---|
| 잠김 | 이전 부위 또는 복원도 조건 필요 |
| 진행 가능 | 선택 가능 |
| 진행 중 | 완료하지 않은 복원 세션 있음 |
| 완료 | 보상 수령 완료 |
| 완전 복원 | 최고 단계 완료 |

---

## 5.6 대표 이미지 변화

유적 상세 화면의 대표 이미지는 전체 복원도에 따라 변화한다.

| 복원도 | 이미지 상태 |
|---|---|
| 0~20% | 먼지, 이끼, 무너진 구조 |
| 21~50% | 주요 구조가 보이기 시작 |
| 51~80% | 문양과 색감이 살아남 |
| 81~99% | 거의 완성 |
| 100% | 반짝이는 완전 복원 상태 |

---

## 5.7 유적 완료 팝업

유적의 클리어 조건을 달성하면 표시한다.

```text
유적 복원 완료!
숲속 고대 사원이 다시 빛나기 시작했어요.

역사 카드가 열렸어요.
보상: 골드 2,500 / 복원 재료 120

[역사 카드 보기] [월드맵으로]
```

---

## 6. 화면 4: 도감 / 역사 카드 화면

## 6.1 역할

복원한 유적의 정보와 짧은 역사 지식을 확인하는 화면이다. 교육 요소와 수집 요소를 담당한다.

핵심 감정:

> 내가 복원한 유적의 이야기를 하나씩 모으고 있다.

---

## 6.2 진입 경로

- 하단 네비게이션에서 도감 선택
- 유적 완료 팝업에서 역사 카드 보기 선택
- 유적 상세 화면에서 카드 보기 선택

---

## 6.3 화면 구성

```text
┌─────────────────────────┐
│ 유적 도감                 │
│ 수집률 3 / 12             │
├─────────────────────────┤
│ [아시아] [인도] [중동] ... │
├─────────────────────────┤
│ 카드 목록                 │
│ ✅ 작은 항구 사당          │
│ ✅ 붉은 벽돌 왕국          │
│ ✅ 안개의 계단 사원        │
│ 🔒 숲속 고대 사원          │
├─────────────────────────┤
│ 선택 카드 상세             │
│ 짧은 역사 설명             │
└─────────────────────────┘
```

---

## 6.4 역사 카드 상세 구성

- 게임 내 유적명
- 실제 영감 유적명
- 지역/나라
- 시대 또는 문명
- 짧은 설명 3줄
- 재미 포인트 1개
- 복원 달성도
- 획득 보상 기록

---

## 6.5 역사 카드 문체 원칙

- 쉽고 짧게 작성
- 너무 교과서처럼 쓰지 않기
- 한 카드당 3~5문장 이하
- 어려운 용어는 피하거나 풀어서 설명
- 실제 사실 기반으로 작성

예시:

```text
안개의 계단 사원
영감: 보로부두르

아주 오래전 자바 섬에 세워진 거대한 계단식 사원이에요.
위로 올라갈수록 여러 조각과 탑을 만날 수 있어요.
냥냥 복원단은 안개 속에 묻힌 계단을 하나씩 되살렸어요.
```

---

## 6.6 잠긴 카드 표시

잠긴 카드는 실루엣과 힌트만 제공한다.

```text
아직 발견하지 못한 유적이에요.
아시아 파트의 네 번째 유적을 복원하면 열려요.
```

---

## 7. 화면 5: 상점 / 보급품 화면

## 7.1 역할

복원에 필요한 재료, 보조 아이템, 광고 보상, 광고 제거 상품을 제공하는 화면이다.

핵심 감정:

> 복원을 계속할 수 있도록 탐험대 지원을 받는다.

---

## 7.2 진입 경로

- 하단 네비게이션에서 상점 선택
- 재료 부족 팝업에서 이동
- 실패 복구 팝업에서 보급품 부족 시 이동

---

## 7.3 화면 구성

```text
┌─────────────────────────┐
│ 탐험 지원소                │
│ 골드 3,200 / 재료 45       │
├─────────────────────────┤
│ 오늘의 무료 보급품          │
│ [받기]                     │
├─────────────────────────┤
│ 탐험대 지원                │
│ 광고 보고 재료 받기         │
│ [지원 받기]                │
├─────────────────────────┤
│ 복원 도구                  │
│ 행운 붓 / 안전 장갑         │
├─────────────────────────┤
│ 후원자 패스                │
│ 광고 제거 + 작은 보너스     │
│ [구매하기]                 │
└─────────────────────────┘
```

---

## 7.4 상품 구분

### 무료 보급품

- 하루 1회 지급
- 소량의 재료와 골드

### 광고 보급품

- 보상형 광고 시청 후 지급
- 하루 제한 가능
- 재료 부족 시 자연스럽게 유도

### 복원 도구

MVP 후보:

| 아이템 | 효과 |
|---|---|
| 행운 붓 | 다음 복원 성공률 소폭 증가 |
| 안전 장갑 | 다음 실패 시 균열 피해 감소 |
| 고양이 간식 | 고양이 응원 연출, 소량 보너스 |

초기에는 아이템 종류를 많이 늘리지 않는다.

### 후원자 패스

- 광고 제거
- 보상형 광고 버튼을 광고 없이 사용 가능하게 처리하는 방향 권장
- 작은 후원 배지 또는 보너스 재료 제공 가능

---

## 7.5 광고 제거 구매자 처리

광고 제거 구매자는 다음 혜택을 받는다.

- 전면 광고 제거
- 보상형 광고 버튼 사용 시 광고 없이 보상 지급
- 후원자 배지 표시

이 처리가 만족감이 높다.

## 8. 공통 하단 네비게이션

MVP에서는 하단 네비게이션을 사용하는 것을 권장한다.

탭 구성:

1. 여행
2. 복원
3. 도감
4. 지원소

유적 상세는 여행 탭 내부 화면으로 처리할 수 있다.

복원 중인 부위가 없을 때 복원 탭을 누르면:

```text
복원할 유적을 먼저 선택해 주세요.
[월드맵으로 가기]
```

## 9. 팝업 목록

MVP에서 필요한 주요 팝업:

3. 광고 복구 제안 팝업
4. 재료 부족 팝업
5. 유적 완료 팝업
6. 역사 카드 획득 팝업
7. 광고 제거 구매 완료 팝업
8. 저장 오류 또는 광고 로드 실패 안내

## 10. 우선 구현 순서

1. 월드맵 화면 기본 구조
2. 유적 상세 화면 기본 구조
3. 메인 복원 화면 기본 구조
4. 복원 성공/실패 흐름
5. 유적 완료 처리
6. 도감/역사 카드 화면
7. 상점/보급품 화면
8. 광고/IAP 연결용 인터페이스 준비
9. 애니메이션/이펙트 강화

---

## 11. MVP 완료 기준

다음 조건을 만족하면 화면 MVP는 완료로 본다.

- 5개 화면 이동이 정상 동작한다.
- 월드맵에서 유적을 선택할 수 있다.
- 유적 상세에서 부위를 선택할 수 있다.
- 복원 화면에서 복원하기/완료하기가 작동한다.
- 성공/실패 결과가 화면에 반영된다.
- 유적 완료 시 도감 카드가 열린다.
- 상점에서 무료 보급품을 받을 수 있다.
- 광고/IAP는 실제 연동 전이라도 서비스 인터페이스가 분리되어 있다.

## 12. 화면 설계 최종 방향

냥냥 유적 복원단은 화면을 많이 늘리는 게임이 아니다. 핵심은 메인 복원 화면의 손맛과 월드맵의 여행감이다.

모든 화면은 다음 질문에 답해야 한다.

> 유저가 지금 무엇을 누르면 되는지 바로 알 수 있는가?

화면이 예뻐도 선택이 복잡하면 안 된다. 5개 화면 안에서 작고 단단한 게임을 완성하는 것을 우선한다.
